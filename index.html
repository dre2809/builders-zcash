02 Curriculum
8 Weeks · 40h

From zero to
shipping on mainnet.

Each week is 5 hours: 3 hours of self-paced material (video + reading) and a 2-hour mandatory lab. Office hours weekly. Live demo day at the end.

01Foundations

Environment & Node Operation

Install and sync zebrad, understand checkpointing, run your first RPC calls. Set up lightwalletd and the Zaino indexer. Learn the shape of the Zcash chain from the node operator's side.

zebrad lightwalletd Zaino Docker
Lab 01
Script that monitors new blocks and triggers an alert on any shielded transaction above N ZEC.
02SDK

Wallet SDK

Work with zcash_client_backend and zcash_primitives. HD derivation, spending and viewing keys, Unified Addresses (ZIP-316): construction and decomposition.

Rust zcash-wallet-sdk ZIP-316 BIP-32
Lab 02
CLI that generates a seed, derives a UA with Orchard + Sapling + transparent receivers, validates addresses.
03Chain

Chain Layer

Deep dive into ZebraRPC: block data, mempool, transaction decoding, balance queries, z_validateaddress, z_listunifiedreceivers. Build the chain data plane for any Zcash app.

JSON-RPC Next.js TypeScript zebrad
Lab 03
Minimal block explorer that shows tx history for a transparent address and validates UAs with receiver breakdown.
04Shielded

Shielded Transactions — Orchard & Halo2

Work with zkool's GraphQL layer: accounts, pools, notes. Build transactions with the pay mutation. Pool selection via bitmask. Memo encryption (512 bytes). Inspect Halo2 proofs — understand what they prove and why.

zkool GraphQL Orchard Halo2
Lab 04
Send 3 shielded TAZ transactions (Orchard → Orchard) with encrypted memos. Verify on-chain, inspect the proof structure.
05Keys

Viewing Keys & Selective Disclosure

FVK, IVK, OVK — what each reveals and to whom. View-only wallets. Payment disclosure (ZIP-310). Audit flows for compliance use cases. Privacy with auditability.

ZIP-310 FVK / IVK / OVK Compliance
Lab 05
View-only wallet that shows received transactions without access to the spending key. Test with a second wallet sending to you.
06Build

Product Build — Kickoff

Choose your track: wallet UI (Tauri or web), payment gateway (server with webhooks), or a shielded tool (CLI or library). Spec + architecture reviewed by mentor. First MVP running against testnet.

Your Stack Architecture Mentor Review
Lab 06
MVP of your chosen project running end-to-end on testnet. Core flow must complete at least one real TAZ transaction.
07Security

Testing & Security

Common pitfalls: replay, reorg handling, pool confusion, memo leakage. Property tests with proptest or fast-check. Fuzz testing for address and transaction parsers. Zcash security checklist derived from public audits.

proptest fast-check Fuzzing Security
Lab 07
Test suite on your week 6 project. Minimum 70% coverage on critical paths. Include at least one property test and one fuzz target.
08Ship

Ship & Contribute

Deploy on mainnet with a small real value ($5–10 in ZEC). Write a technical blog post ("how I built X"). Open a PR in an official Zcash repository. Live demo day: 15 minutes per participant.

Mainnet Deploy Blog Post GitHub PR Demo Day
Lab 08
Three deliverables: running mainnet project + published technical post + PR link to zebra, zkool2, zcash-wallet-sdk, zallet, or zcash-migrate.
03 Quickstart
Run Locally

Your first shielded
transaction in 10 minutes.

Run the full Zcash developer stack on your own machine. Copy, paste, ship.

builders.z.cash — starter
# 1. Clone the starter repo
git clone https://github.com/builders-zcash/starter
cd starter

# 2. Start zebrad (testnet) + zkool + lightwalletd
docker-compose up -d

# 3. Wait for first sync (checkpointed, ~5 min on good connection)
docker-compose logs -f zebrad

# 4. Create a testnet account
curl -X POST http://localhost:8001/graphql \
  -H "Content-Type: application/json" \
  -d '{"query":"mutation{createAccount(name:\"dev\",seedPhrase:\"\",pools:7)}"}'

# 5. Get your Unified Address
curl http://localhost:8001/graphql \
  -d '{"query":"{addressByAccount(id:1){ua}}"}'

# 6. Request TAZ from the faucet
curl -X POST https://faucet.builders.z.cash/drip \
  -H "Authorization: Bearer $GH_TOKEN" \
  -d '{"address":"utest1..."}'

# 7. You now have shielded TAZ. Welcome to Zcash.
// Cargo.toml
[dependencies]
zcash_client_backend = "0.16"
zcash_primitives = "0.21"
zcash_address = "0.6"

// src/main.rs — generate a Unified Address
use zcash_client_backend::keys::UnifiedSpendingKey;
use zcash_primitives::consensus::Network;

fn main() -> anyhow::Result<()> {
    let seed = [0u8; 32]; // replace with real seed
    let usk = UnifiedSpendingKey::from_seed(
        &Network::TestNetwork,
        &seed,
        zip32::AccountId::ZERO,
    )?;

    let ufvk = usk.to_unified_full_viewing_key();
    let (ua, _) = ufvk.default_address(Default::default())?;

    println!("UA: {}", ua.encode(&Network::TestNetwork));
    Ok(())
}

// send-shielded.ts — send shielded TAZ via zkool
import { GraphQLClient, gql } from 'graphql-request';

const client = new GraphQLClient(
  'https://zkool-testnet.builders.z.cash/graphql'
);

const PAY = gql`
  mutation Pay($accountId: Int!, $recipients: [RecipientInput!]!) {
    pay(accountId: $accountId, recipients: $recipients, srcPools: 4)
  }
`;

const result = await client.request(PAY, {
  accountId: 1,
  recipients: [{
    address: 'utest1...',
    amount: '0.1',
    memo: 'First shielded tx',
  }],
});

console.log('txid:', result.pay);

# send_shielded.py
import requests

ZKOOL = "https://zkool-testnet.builders.z.cash/graphql"

def gql(query, variables=None):
    r = requests.post(ZKOOL, json={"query": query, "variables": variables})
    r.raise_for_status()
    return r.json()["data"]

# 1. Create account (pools=7: transparent + sapling + orchard)
account = gql('''
  mutation { createAccount(name:"dev", seedPhrase:"", pools:7) }
''')["createAccount"]

# 2. Get unified address
addr = gql(f"""
  {{ addressByAccount(id:{account}) {{ ua }} }}
""")["addressByAccount"]["ua"]

print(f"Your UA: {addr}")

# 3. Send shielded TAZ
tx = gql(f"""
  mutation {{
    pay(
      accountId: {account},
      recipients: [{{ address: "utest1...", amount: "0.1", memo: "hi" }}],
      srcPools: 4
    )
  }}
""")["pay"]

print(f"txid: {tx}")

Outcomes

What graduates ship.
